Express Preferences

You can change some global preferences of JSDAI Express development by going to Window -> Preferences -> JSDAI -> Express dialog window (see Figure 1):
  • Select an option in the Switching to the Express perspective when an Express file is opened or an Express project is created layout:
    • ask - the user is provided with the choices to ask, to switch without asking, or to stay in the current perspective. If the ask option is chosen (default), then the confirmation dialog appears when it is appropriate to switch to the Express perspective. The dialog itself has a check box Remember my decision. So this preference here may be changed from that dialog. But the existence of this preference allows the user to change his decision later.
    • switch to the Express perspective - the Express perspective will be switched on after an Express file is opened or an Express project is created.
    • leave the current perspective - remains the current perspective.
  • You also can set an applicable option below:
    • Delete all existing old JSDAI problem markers when running the express compiler or validation. The below options are disabled after this one is selected.
    • When running the express compiler, delete also unrelated JSDAI Express problem markers.
    • When running validation, delete also unrelated JSDAI P21 problem markers.
