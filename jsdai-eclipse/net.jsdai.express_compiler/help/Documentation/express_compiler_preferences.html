Express Compiler Global Preferences


You can change some global input-output settings for Express Compiler by going to Window -> Preferences -> JSDAI -> Express compiler dialog window (see Figure 1):
  • Input of express files for the compiler:
    • the list file - compile express files in the EXL file list;
    • the Express directory (non-recursively) - the express files are taken from the directory of express files, inner subdirectories are ignored;
    • the Express directory (recursively) -  the express files are taken not only from the directory of express files, but also from its inner subdirectories.
  • Using exclusion list with directory input:
    • use exclusion list - if this box is checked, the express compiler attempts to open ..._excluded.exl file in the root of the project. If the file is found and contains a list of express files with full paths, these files are excluded from the current compilation.
          Note.  If the list file radio button is selected this option is disabled.
  • Creating the library JAR:
    • yes, generate and compile java and create as well as exd  - if this box is checked the JAR and EXD files are created after the compiling of Express file is performed;
    • no, just check for express errors and, if success, create exd only - if this box is checked only EXD file is created after the compiling of Express file is performed.



Figure 1. Preferences dialog window, Express compiler panel
Note.    You can set your own  settings for an Express compiler while creating a new Express project or by going to already created Express project preferences (right click on the project and select Properties -> Express -> Input - output).
